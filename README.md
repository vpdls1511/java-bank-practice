# 기능 요구사항
1. 사용자는 계좌를 생성할 수 있다
2. 사용자는 금액을 입금할 수 있다
3. 사용자는 금액을 출금할 수 있다
4. 사용자는 보유한 계좌를 확인할 수 있다.

# 규칙
Main 클래스에는 비즈니스 로직을 작성하지 않고, 역할은 아래 두 가지로 제한한다.

* 사용자 입력 받기
* 필요한 객체의 메서드 호출

계좌 생성, 입금, 출금, 조회와 같은 실제 기능은 각각 책임을 가진 클래스가 수행하도록 설계하는 것을 목표로 한다.
